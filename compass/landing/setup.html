Outcomes, tied to agents

Agents get hired on a promise. Compass holds them to it.

Every agent in your fleet was brought in to deliver something — clean records, deflected tickets, drafted replies. Compass ties each agent to that outcome and verifies delivery in your own systems, run after run.

~10 min of setup per agent You answer economics, never mechanics Value read from your system of record — never agent self-reports
The idea, in three beats

How an agent becomes accountable.

Before you click anything — this is the whole mental model. The walkthrough below just makes it real.

01 · THE PLAN

An agent is hired like an employee.

It has a job and a number it was brought in to deliver. Most teams never write that number down. Compass makes it the plan — the line every result is measured against.

02 · THE READ

Compass reads the agent. Nobody fills in a form.

From the agent's own telemetry: what it runs on, which tools it calls, what it costs, whose identity it acts under. You confirm — you don't type.

03 · THE LEDGER

Delivery is verified where the work lands.

Value is read from your CRM, helpdesk, or billing — run after run, not at review time. Never from what the agent says about itself.
